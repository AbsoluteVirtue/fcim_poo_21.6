# Потоки

## Польза генераторов и ленивых вычислений
Итераторы -- это пример применения ленивых вычислений на коллекциях. Для того, чтобы обработать один элемент коллекции, не нужно знать об остальных элементах. Итератор может найти следующий элемент, поэтому алгоритму достаточно взаимодействовать с итератором. Сортировка -- пример вычислений, которые требуют знания всей коллекции, так как все элементы нужно будет сравнить друг-с-другом.
Ленивая обработка позволяет свести алгоритмы, которые ее поддерживают, к линейному времени. Так, алгоритм, который фильтрует коллекцию по какому-то критерию, затем применяет какую-то операцию к каждому элементу, а затем вычисляет какое-то значение, используя оставшиеся элементы коллекции, может сделать все эти операции за один проход, собрав с помощью композиции и частичной аппликации цепочку вычислений, и применив ее на все элементы коллекции по очереди.
